# RBCE (Role-Based Collaborative Environment)

**RBCE** — это веб-платформа для управления студенческими проектами с ролевой моделью. Приложение позволяет создавать проекты, управлять участниками, оставлять комментарии и отслеживать прогресс совместной работы.

## 1. Общие сведения

**Наименование программы:** RBCE (Role-Based Collaborative Environment)

**Тип программы:** веб-приложение для управления студенческими проектами.

**Назначение программы:** платформа предназначена для регистрации пользователей, создания и управления проектами, взаимодействия участников через комментарии с отображением ролей (Organizer, Participant, Undefined).

**Область применения:** учебные проекты, командная разработка, студенческие инициативы и любые совместные проекты с ролевой моделью.

## 2. Назначение разработки

### 2.1 Функциональное назначение

Программа обеспечивает:

- регистрацию и авторизацию пользователей (JWT-токены);
- создание проекта (тема, описание, тип, локация, дата);
- просмотр списка всех проектов;
- просмотр деталей проекта (участники с ролями, комментарии);
- присоединение / выход из проекта;
- комментирование (автор + его роль в проекте);
- редактирование / удаление проекта (только для организатора).

### 2.2 Эксплуатационное назначение

Программа предназначена для работы через браузер. Пользователи взаимодействуют с интерфейсом веб-приложения, а сервер на Go обрабатывает запросы, сохраняет данные в SQLite и управляет доступом через JWT-токены.

## 5. Функциональные требования

### 5.1 Регистрация и авторизация

Программа предоставляет страницы:

- `/register` — регистрация пользователя
- `/login` — вход пользователя

При регистрации пользователь вводит:
- email
- пароль
- имя пользователя
- полное имя
- информацию о себе

После успешного входа создаётся **JWT-токен**, который сохраняется в `localStorage`. Токен передаётся в заголовке `Authorization: Bearer <token>`.

Доступ к защищённым страницам и API разрешён только авторизованным пользователям.

### 5.2 Управление проектами

Программа предоставляет следующие возможности:

- **Создание проекта** — пользователь автоматически становится организатором (Organizer)
- **Просмотр списка проектов** — отображаются все активные проекты
- **Просмотр деталей проекта** — участники с ролями, комментарии
- **Присоединение / выход** — пользователь может присоединиться к проекту (роль Participant) или выйти из него
- **Редактирование / удаление** — доступно только организатору проекта

### 5.3 Комментарии

- Комментарии отображают автора и его роль в проекте
- Роли определяются автоматически:
  - **Organizer** — организатор проекта
  - **Participant** — участник, присоединившийся к проекту
  - **Undefined** — пользователь, не присоединившийся к проекту, но оставивший комментарий

### 5.4 Soft Delete

При удалении проекта используется **soft delete** — проект помечается флагом `is_voided = 1` и не отображается в списке, но сохраняется в базе данных.

### 6.1 Аутентификация

| Метод | Эндпоинт | Описание | Требует JWT |
|-------|----------|----------|-------------|
| POST | `/api/Auth/Register` | Регистрация пользователя | нет |
| POST | `/api/Auth/login` | Вход пользователя | нет |

### 6.2 Проекты (Topics)

| Метод | Эндпоинт | Описание | Требует JWT |
|-------|----------|----------|-------------|
| GET | `/api/Topics/GetTopics` | Получение списка проектов | да |
| GET | `/api/Topics/GetTopic?id={id}` | Получение деталей проекта | да |
| POST | `/api/Topics/CreateTopic` | Создание проекта | да |
| PUT | `/api/Topics/UpdateTopic?id={id}` | Обновление проекта (только Organizer) | да |
| DELETE | `/api/Topics/DeleteTopic/{id}` | Удаление проекта (только Organizer) | да |
| POST | `/api/Topics/JoinLeaveTopic?id={id}` | Присоединение/выход из проекта | да |

| Метод | Эндпоинт | Описание | Требует JWT |
|-------|----------|----------|-------------|

### 7. Логика работы

### 7.1 Регистрация и аутентификация
1. Пользователь отправляет запрос на `/api/Auth/Register` с email, паролем и данными профиля.
2. Сервер проверяет уникальность email и username.
3. Пароль хешируется с помощью SHA-256.
4. Пользователь сохраняется в базу данных.
5. Сервер генерирует JWT-токен и возвращает его клиенту.
6. Клиент сохраняет токен в localStorage.

### 7.2 Создание проекта
1. Авторизованный пользователь отправляет POST-запрос с данными проекта.
2. Сервер создаёт проект и автоматически добавляет создателя как организатора (role = "Organizer").
3. Проект сохраняется в базу данных.

### 7.3 Управление участниками
- **Присоединение:** пользователь отправляет запрос на `/api/Topics/JoinLeaveTopic`. Сервер добавляет его с ролью "Participant".
- **Выход:** при повторном запросе сервер удаляет пользователя из участников проекта.

### 7.4 Комментарии
1. Пользователь пишет комментарий.
2. Сервер определяет роль пользователя в проекте:
   - Если пользователь — организатор → role = "Organizer"
   - Если пользователь — участник → role = "Participant"
   - Если пользователь не связан с проектом → role = "Undefined"
3. Комментарий сохраняется вместе с ролью на момент написания.

### 7.5 Удаление проекта (Soft Delete)
- Только организатор может удалить проект.
- Проект не удаляется физически из БД, а помечается флагом `is_voided = 1`.
- Удалённые проекты не отображаются в списке.
